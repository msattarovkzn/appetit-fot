## 1. Назначение системы

Внутренняя система учёта рабочего времени, ФОТ и аналитики для ресторанной сети «Аппетит».

**Функции:**
- Открытие/закрытие смен сотрудниками через PIN
- Закрытие кассовой смены с вводом выручки
- Расчёт ФОТ по категориям (кухня, администрация, техперсонал и др.)
- Контроль ФОТ % (норма: общий <27.5%, кухня <14.5%)
- Проверка дней бухгалтером (RED/YELLOW/GREEN)
- График смен (план vs факт)
- Аналитика и тренды
- Личный кабинет сотрудника

## 8. Бизнес-процессы

### 8.1 Рабочий день сотрудника
```
Сотрудник → /shift → выбирает филиал → вводит PIN
  → Открыть смену (opened_at = NOW, status=open)
  → Работает...
  → Закрыть смену (closed_at = NOW, status=closed,
                    total_minutes вычисляется автоматически)
```

### 8.2 Закрытие кассовой смены
```
Кассир → /cashier → вводит PIN → открывает смену (если не открыта)
  → В конце смены: вводит выручку + заказы + выносы → Закрыть день
  → Создаётся CashierSession (сессия кассира)
  → BranchDailyReport создаётся или обновляется (суммируется)
  → calculate_payroll_for_day() — пересчёт PayrollEntry для всех закрытых смен
  → Создаётся FotSummary
  → Telegram уведомление
```

**Мультикассир:** каждый кассир закрывает свою сессию. Выручка суммируется.

### 8.3 Расчёт ФОТ

```
total_minutes = closed_at - opened_at (в минутах)
approved_hours = total_minutes / 60  (или override от менеджера)

Почасовой: base_pay = (rate / 60) * total_minutes
Фикс/день: base_pay = fixed_daily_rate

Бонус кассира-администратора:
  Пн–Чт, Вс: orders_count × 7 ₽
  Пт, Сб:    orders_count × 5 ₽

ФОТ общий %  = (sum(total_pay)) / revenue × 100
ФОТ кухни %  = (sum(kitchen_pay)) / revenue × 100

Статус total: GREEN <27.5%, YELLOW 27.5-29%, RED >29%
Статус кухни: GREEN <14.5%, YELLOW 14.5-15.5%, RED >15.5%
```

**Ставка берётся на дату смены** (history-based, из employee_rates).

### 8.4 Проверка дней бухгалтером

```
Каждое утро: /admin/review → выбрать вчерашнюю дату
  → Для каждого филиала система вычисляет issues:
      day_not_closed     — нет BranchDailyReport
      unclosed_shifts    — есть open shifts
      anomaly_warning    — смены 14-16 ч (нерешённые)
      anomaly_critical   — смены >16 ч (нерешённые)
      extra_shifts       — есть доп. смены
      manual_corrections — бухгалтер правил часы вручную

  Статус: RED (есть issues) → YELLOW (нет issues, не проверен)
         YELLOW → GREEN (бухгалтер нажал "Проверка завершена")

  Действия по смене: закрыть | изменить время | изменить часы |
                     разовая ставка | аннулировать
```

### 8.5 Дополнительная смена
```
Сотрудник пытается открыть → "already_had_shift" → нельзя
Кассир: /cashier → "Открыть доп. смену" → выбор сотрудника +
        время начала + причина → смена с is_extra_shift=True
Закрытие: кассир через /cashier → "Закрыть смену"
```

## 11. Тестовый режим

- `ENVIRONMENT=staging` → включает `/test/*` endpoints на backend
- `NEXT_PUBLIC_TEST_MODE=true` → показывает UI выбора даты/часов на `/shift` и `/cashier`
- `NEXT_PUBLIC_LIVE_TEST_MODE=true` → скрывает тестовый UI, показывает "Реальный тест"
- `/test` страница — прямой доступ к тестовым операциям (только branch_id=1)

## 12. Уведомления

При каждом закрытии дня (`close-day-by-pin`) отправляется Telegram-сообщение с:
- Название филиала
- Дата
- Выручка
- Заказы / Выносы
- Незакрытые смены (предупреждение)
- ФОТ %

Хранятся в таблице `notifications` со статусом sent/failed.
